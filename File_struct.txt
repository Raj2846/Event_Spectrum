### 📁 **events/** (Main Application)

#### `models.py` - Database Tables
```python
# Two main models:
1. Event - Stores event details
2. Ticket - Stores bookings & QR codes
```

#### `views.py` - Business Logic
```python
# Main functions:
- event_list() → Show all events
- event_detail() → Show event & booking form
- payment_success() → Handle Razorpay callback
- scan_qr() → Verify QR codes
- admin_dashboard() → Admin stats
```

#### `forms.py` - Input Validation
```python
# TicketBookingForm with validation for:
- Full name (letters only, 3+ chars)
- Email (valid format)
- Phone (10-digit Indian number)
```

#### `admin.py` - Django Admin Panel
```python
# Admin interface for:
- Creating/editing events
- Viewing tickets
- Exporting data to CSV
```

#### `urls.py` - App URLs
```python
# Routes:
- / → Event list
- /event/<id>/ → Event detail
- /payment/success/ → Payment callback
- /scan/ → QR scanner
- /dashboard/ → Admin dashboard
```

## 📝 Setup Commands in Order

```bash
# 1. Create project structure
mkdir event_management
cd event_management

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create Django project
django-admin startproject event_project .

# 5. Create app
python manage.py startapp events

# 6. Create directories
mkdir -p templates/events
mkdir -p static/css static/js static/images
mkdir -p media/qr_codes media/event_images

# 7. Copy all the code files I provided into their respective locations

# 8. Create MySQL database
mysql -u root -p
CREATE DATABASE event_db;
exit

# 9. Run migrations
python manage.py makemigrations
python manage.py migrate

# 10. Create superuser
python manage.py createsuperuser

# 11. Run server
python manage.py runserver
```
